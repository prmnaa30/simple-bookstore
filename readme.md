# 🚀 [Nama Project Kamu]

> Sebuah project eksperimental untuk mengimplementasikan metode **Mini Sprint** dalam pengembangan *software* berskala kecil.

## 📌 Tentang Project Ini

Project ini dibangun dengan tujuan utama untuk menerapkan siklus pengembangan yang cepat (*rapid development cycle*) menggunakan konsep **Mini Sprint**. Daripada menggunakan siklus sprint tradisional yang memakan waktu 2-4 minggu, project ini dipecah menjadi iterasi-iterasi singkat berdurasi `[1-3 Hari/1 Minggu]` untuk memastikan *delivery* fitur yang cepat dan evaluasi yang konstan.

**Tujuan Utama:**
* [Contoh: Membangun MVP dari aplikasi *task tracker*]
* Mengimplementasikan manajemen waktu yang efisien.
* Melakukan evaluasi dan iterasi secara berkala di setiap akhir mini sprint.

## ⏱️ Mini Sprint Workflow

Project ini berjalan dalam ritme sprint yang dipercepat:
1. **Sprint Planning (Singkat):** Menentukan 1-3 *task* krusial untuk sprint ini.
2. **Deep Work / Execution:** Fokus penuh pada *development* tanpa distraksi.
3. **Sprint Review & Retrospective:** Mengecek hasil, merapikan *code*, dan mencatat apa yang bisa diperbaiki untuk sprint selanjutnya.
